# 🟢 LEVEL 5 — Recursion Foundations

- Recursion is one of the **most important ideas before Dynamic Programming**.

- Dynamic Programming is actually:

```text
Recursion + remembering answers
```

- So first we must understand **how recursion works**.

---

## 1️⃣ What is Recursion?

- Recursion means:

> A function solving a problem by **calling itself on a smaller version of the same problem**.

- Think of it like asking the **same question but easier each time**.

---

## Example Idea

- Imagine a line of people.

- You ask the first person:

> "How many people are behind you?"

- That person asks the next person the same question.

- The chain continues until the last person answers:

> "0 people behind me."

- Then answers propagate backward.

- That process is **recursion**.



---

## 2️⃣ Example: Counting Down

- Suppose we want to print numbers:

```text
5 4 3 2 1
```

- **Recursive idea:**

- **print(n):**
- **then print(n-1):**
- **then print(n-2):**
- **...:**

- **Code idea:**

```cpp
void countDown(int n)
{
    if(n == 0)
        return;

    cout << n;

    countDown(n-1);
}
```

- **Call:**

```cpp
countDown(5);
```

- **Output:**

```text
5 4 3 2 1
```

- The function **calls itself with a smaller number**.

---

## 3️⃣ The Two Essential Parts of Recursion

- Every recursive function must have **two parts**.

---

## Base Case

- The **stop condition**.

- **Example:**

- **if n == 0 → stop:**

- Without a base case, recursion **never stops**.

---

## Recursive Case

- The part where the function **calls itself**.

- **Example:**

- **countDown(n-1):**

- This reduces the problem size.

---

## 4️⃣ Recursive Thinking

- Recursion works by thinking:

> "Assume the smaller problem is already solved."

- **Example:**

- Sum numbers from **1 to n**.

- Instead of adding everything manually:

```text
1 + 2 + 3 + ... + n
```

- **Think like this:**

- $sum(n) = n + sum(n-1)$

- **Example:**

- $sum(5)$
- $= 5 + sum(4)$
- $= 5 + 4 + sum(3)$
- $= 5 + 4 + 3 + sum(2)$

- **Eventually:**

- $sum(1) = 1$

- Now answers build back up.

---

## 5️⃣ Recursion Tree

- Recursive calls form a **tree structure**.

- **Example:**

```text
sum(3)
```

- **Calls:**

```text
sum(3)
  |
sum(2)
  |
sum(1)
```

- Each call waits for the smaller call to finish.

- Then returns the result.



---

## 6️⃣ The Call Stack

- Computers remember recursive calls using a **stack**.

- Think of stacking plates.

- **Example call order:**

- **sum(3):**
- **sum(2):**
- **sum(1):**

- **Stack looks like:**

```text
sum(3)
sum(2)
sum(1)
```

- **Now returning happens:**

- **sum(1) → returns 1:**
- **sum(2) → returns 3:**
- **sum(3) → returns 6:**



---

## 7️⃣ Recursive Example: Factorial

- Factorial means:

- $5! = 5 \times 4 \times 3 \times 2 \times 1$

- **Recursive definition:**

- $n! = n \times (n-1)!$

- **Code example:**

```cpp
int fact(int n)
{
    if(n == 1)
        return 1;

    return n * fact(n-1);
}
```

- **Call:**

```cpp
fact(5)
```

- **Execution:**

- **5 * fact(4):**
- **5 * 4 * fact(3):**
- **5 * 4 * 3 * fact(2):**
- **5 * 4 * 3 * 2 * fact(1):**

- **Base case:**

- **fact(1) = 1:**

- **Final result:**

```text
120
```

---

## 8️⃣ Why Recursion Is Powerful

- Recursion helps solve problems that are naturally **self-similar**.

- **Examples:**

- tree traversal
- divide-and-conquer algorithms
- dynamic programming
- backtracking

- Many problems become **simpler to describe recursively**.

---

## 9️⃣ Problem with Pure Recursion

- Recursion can repeat the same work many times.

- **Example:**

- Fibonacci numbers.

- **Definition:**

- $F(n) = F(n-1) + F(n-2)$

- **Recursion tree:**

```text
F(5)
├── F(4)
│   ├── F(3)
│   └── F(2)
└── F(3)
```

- **Notice:**

- **F(3):**

- is computed **twice**.

- For large n, this becomes extremely slow.



---

## 🔟 Why DP Improves Recursion

- Dynamic Programming fixes recursion's biggest problem.

- Instead of recomputing:

- **F(3):**

- we **store the answer**.

- Then reuse it.

- This is called:

- **memoization:**

- **So:**

- **DP = recursion + memory:**

---

## 🧠 Recursion Mental Model

- When solving recursive problems ask:

- **1. What is the smallest version of the problem?:**
- **2. What is the base case?:**
- **3. How does the problem shrink?:**
- **4. How do results combine?:**

- If these answers are clear, recursion becomes easy.

---

## 🧭 Why Level 5 Matters for Grid DP

- Grid DP problems often start as recursive problems like:

- $maxPath(r,c) = value[r][c] + \max(maxPath(r-1,c), maxPath(r,c-1))$

- DP is simply **the optimized version of this recursion**.
