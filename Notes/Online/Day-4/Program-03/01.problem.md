# 📌 Problem Statement

- You are given a binary matrix of size N × M.

- You may perform the following operations any number of times:
- - Flip any entire row (convert all 0 → 1 and 1 → 0)
- - Flip any entire column

- After performing any sequence of flips, your goal is to:

- 👉 Maximize the number of rows that become **identical**

---

## 🎯 Objective

- Return the **maximum number of rows that can be made identical** after optimal flips.

---

## 📥 Input Format

- - First line: two integers N and M  
- - Next N lines: M binary integers (0 or 1)

---

## 📤 Output Format

- - A single integer → maximum number of identical rows

---

## 🧠 Constraints

- - 1 ≤ N, M ≤ 1000  
- - grid[i][j] ∈ {0, 1}

---

## ⚠️ Important Clarification

- - Flipping a row toggles all elements in that row
- - Flipping a column toggles all elements in that column
- - Flips can be applied in any order and any number of times

---

## 🧪 Sample Inputs & Outputs

### Sample 1

```text
1 0 1
0 1 0
1 0 1
```

- **Output:**
- 3

---

### Sample 2

```text
1 1 0
0 0 1
1 1 0
```

- **Output:**
- 3

---

## 🤔 Edge-Case Thinking Prompts

- - Can two different rows become equal after flips?
- - What happens if one row is the complement of another?
- - Does the order of flips matter?
- - Can flipping columns align multiple rows simultaneously?

---

## 🧠 Thinking Like a Programmer

### What must be tracked?
- - Row patterns and their equivalence

### What can change?
- - Bit values due to flips

### What must never change?
- - Relative structure of bits within a row

### What brute force would do?
- - Try all flip combinations → exponential

---

- - No code
