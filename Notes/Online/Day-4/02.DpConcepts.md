# 🟢 LEVEL 0 — Absolute Thinking Foundations

- These are not programming ideas yet.
- They are thinking tools.
- Without them, students cannot understand grids or DP states.

## 1️⃣ Counting & Basic Arithmetic

### What is Counting?

- Counting means knowing how many things exist.
- **Example:**

- 🍎 🍎 🍎
- How many apples?
- We count:

- 1 → 2 → 3
- **Answer:**
- 3
- Counting helps us measure size.

### Addition (Putting Things Together)

- Addition means combining groups.
- **Example:**

- 2 apples + 3 apples
- **Visual:**

- 🍎 🍎   +   🍎 🍎 🍎
- Combine them:

- 🍎 🍎 🍎 🍎 🍎
- **Answer:**

- 2 + 3 = 5

- **Why this matters later:**
- Algorithms often combine results from smaller parts.
- **Example in DP:**

- `path[r][c] = left + top`
- That is addition of solutions.

### Subtraction (Removing Things)

- Subtraction means taking some away.
- **Example:**

- 5 apples − 2 apples
- **Visual:**

- 🍎🍎🍎🍎🍎
- remove 🍎🍎
- **Remaining:**

- 🍎🍎🍎
- **Answer:**

- 5 − 2 = 3

- **Why this matters later:**
- Many algorithms compute difference between positions.
- **Example:**

- `distance = right - left`

### Comparison (Which Is Bigger?)

- We compare numbers to know order.
- **Example:**

- 5 > 3
- **Means:**
- 5 is bigger than 3.
- **Example:**

- 2 < 7
- **Means:**
- 2 is smaller than 7.

- **Why important?**
- Many algorithms ask:

- max(a,b)
- min(a,b)
- **Example in grid DP:**

- `best = max(top, left)`
- We choose better option.

## 2️⃣ Coordinates in a Grid

- Now imagine a chessboard.

- □ □ □
- □ □ □
- □ □ □
- To describe a square we use coordinates.
- Like a map.

### Row

- Rows go horizontally.
- **Example:**

- Row 0 → □ □ □
- Row 1 → □ □ □
- Row 2 → □ □ □

### Column

- Columns go vertically.
- **Example:**

- Col0 Col1 Col2
-  □    □    □
-  □    □    □
-  □    □    □

### Cell Position

- Each square has a coordinate:

- (row, column)
- **Example:**

- (1,2)
- **Means:**
- Row = 1
- Column = 2
- **Visual:**

- Row 0: □ □ □
- Row 1: □ □ X
- Row 2: □ □ □
- X is at (1,2).

### Why This Matters

- Every grid problem stores values like this:

- `grid[r][c]`
- **Meaning:**

- row r
- column c
- DP tables use exactly this structure.

## 3️⃣ Directions

- In grid problems, movement matters.
- Think of walking on tiles.

### Four Basic Directions

-      ↑
- ←   X   →
-      ↓
- **Meaning:**

| Direction | Change |
| --- | --- |
| Up | row - 1 |
| Down | row + 1 |
| Left | column - 1 |
| Right | column + 1 |

- **Example:**
- From position (2,3):

| Move | New Position |
| --- | --- |
| Up | (1,3) |
| Down | (3,3) |
| Left | (2,2) |
| Right | (2,4) |

### Diagonal Directions

- Sometimes movement is diagonal.

- ↖   ↗
-    X
- ↙   ↘
- **Example:**
- From (2,2):

| Move | Position |
| --- | --- |
| ↖ | (1,1) |
| ↗ | (1,3) |
| ↙ | (3,1) |
| ↘ | (3,3) |

- **Example from your DP puzzles:**
- Rainwater flows only:

- ↙
- ↘
- So cells depend only on diagonal neighbors.

## 4️⃣ Neighbors

- A neighbor is a cell next to another cell.
- Imagine sitting in a classroom.
- Your neighbors are the students around you.
- **Example Grid**
- 1 2 3
- 4 5 6
- 7 8 9
- Cell 5 has neighbors.
- **Direct neighbors**
- 2
- 4 5 6
- 8
- These are up, down, left, right.
- **Diagonal neighbors**
- 1   3
-  5
- 7   9
- These are diagonal.

### Why This Matters for DP

- Most grid DP problems say:

- A cell depends on its neighbors.
- **Example:**

- `dp[r][c] = best path from neighbor`
- So understanding neighbors is critical.

---

## 🧠 Mental Model Students Must Build

- Before coding anything, students must see grids like this:

- each cell = a little problem
- neighbors = smaller problems
- **Example mental model:**

- `dp[r][c] =`
- solution using dp of neighbors
- This idea becomes the core of Dynamic Programming.
