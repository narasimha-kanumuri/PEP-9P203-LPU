- Find longest increasing path(right, down)
- Start at (0,0) = 1
- Go right to (0,1) = 2
- Go right to (0,2) = 3
- Go down to (1,2) = 4
- Go down to (2,2) = 5
- Longest path is 5

- **Brute force** would check all paths, which is inefficient.
- **Dynamic programming** stores results of subproblems to avoid redundant calculations.

## Problem

### Repetition

Take one cell at (1,1) = 3

- **Question:** If i reach this point from different paths, will further paths from here be the same or change?
- **Answer:** Yes, because the longest path from (1,1) will always be the same regardless of the path taken to reach it.
- **Possible paths to (1,1):**
- From (0,1) = 2 → (1,1) = 3 
- From (1,0) = 2 → (1,1) = 3
- **Longest path from (1,1) is always 3 → 4 → 5**
- This is why we can use memoization to store the longest path from (1,1) and reuse it whenever we reach that cell again.

- **Memoization**: Store results of longest path from each cell.
- When we reach a cell, check if we already computed the longest path from there. If yes, return it. If no, compute it and store it for future use.
- This way, we avoid redundant calculations and improve efficiency.

- The longest path in the entire grid is 5, starting from (0,0).
- By using memoization, we can efficiently compute the longest path from any cell without redundant calculations.

## Example

```cpp
int longestIncreasingPath(vector<vector<int>>& matrix) {
    if (matrix.empty()) return 0;
    int m = matrix.size(), n = matrix[0].size();
    vector<vector<int>> memo(m, vector<int>(n, 0));
    int longest = 0;

    function<int(int, int)> dfs = [&](int r, int c) {
        if (memo[r][c] != 0) return memo[r][c];
        int maxLength = 1; // At least the cell itself
        vector<pair<int, int>> directions = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};
        for (auto& dir : directions) {
            int nr = r + dir.first, nc = c + dir.second;
            if (nr >= 0 && nr < m && nc >= 0 && nc < n && matrix[nr][nc] > matrix[r][c]) {
                maxLength = max(maxLength, 1 + dfs(nr, nc));
            }
        }
        return memo[r][c] = maxLength;
    };

    for (int i = 0; i < m; ++i) {
        for (int j = 0; j < n; ++j) {
            longest = max(longest, dfs(i, j));
        }
    }
    return longest;
}
```

## What exactly is memoization?

- **Memoization** is a technique used in dynamic programming to store the results of expensive function calls and return the cached result when the same inputs occur again.
- In the context of our longest increasing path problem, we use a 2D vector `memo` to store the longest path length starting from each cell.


## What exactly are we storing in the memoization table?

- In our example, the memoization table `memo` is a 2D vector where `memo[i][j]` stores the length of the longest increasing path starting from the cell `(i, j)`.
- Initially, all values in `memo` are set to 0, indicating that no cell has been computed yet.
- When we compute the longest path from a cell `(i, j)`, we store the result in `memo[i][j]`.
- If we encounter the same cell `(i, j)` again, we can directly return the stored value from `memo[i][j]` instead of recomputing it.

## State - Definition, Transition, Base Case, Answer

- **State**: The state represents the parameters that define a subproblem. In our longest increasing path problem, the state can be defined as the current cell `(i, j)` we are at.
- **Transition**: The transition defines how to move from one state to another. In our problem, we can move to adjacent cells (up, down, left, right) if the value in the adjacent cell is greater than the current cell's value.
- **Base Case**: The base case is the simplest instance of the problem that can be solved directly. In our problem, the base case occurs when we reach a cell that has no adjacent cells with a greater value, in which case the longest path from that cell is 1 (the cell itself).
- **Answer**: The answer is the final result we want to obtain. In our problem, the answer is the length of the longest increasing path in the entire grid, which we can find by iterating through all cells and taking the maximum value from the memoization table.
- By defining the state, transition, base case, and answer, we can systematically approach the problem and implement a dynamic programming solution using memoization.

## Dependency Direction

- **Question**: dp[i][j] depends on which cells?
- **Answer**: dp[i][j] depends on its adjacent cells (i-1, j), (i+1, j), (i, j-1), and (i, j+1) if those cells have a greater value than dp[i][j].
- This means that the longest path from cell (i, j) can only be computed after we have computed the longest paths from its adjacent cells that have greater values.
- Therefore, the dependency direction is from the adjacent cells to the current cell (i, j). We need to compute the longest paths from the adjacent cells first before we can compute the longest path from (i, j).


## Summary of terminology:

- **State**: The parameters that define a subproblem (e.g., current cell).
- **Transition**: How to move from one state to another (e.g., moving to adjacent cells).
- **Base Case**: The simplest instance of the problem that can be solved directly (e.g., a cell with no greater adjacent cells).
- **Answer**: The final result we want to obtain (e.g., length of the longest increasing path in the grid).
- **Dependency Direction**: The order in which we need to compute the states based on their dependencies (e.g., from adjacent cells to the current cell).
