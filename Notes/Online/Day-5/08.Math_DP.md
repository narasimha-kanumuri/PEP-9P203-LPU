# 🟢 LEVEL 8 — Supporting Mathematical Tools for DP

---

## Concepts

- Grid DP problems sometimes need **small mathematical tricks**.
- These tricks help us compute answers **much faster**.
- **The most important tools are:**
- prefix sums
- parity (odd / even)
- toggling values
- fast submatrix calculations

---

## 1️⃣ Prefix Sum (Running Total)

- **Prefix sum means:**
- Store the total of numbers **from the beginning up to a position**.
- **Example array:**

> ```text
> 2 4 1 3
>  ```

- **Step by step:**
- $prefix[0] = 2$
- $prefix[1] = 2 + 4 = 6$
- $prefix[2] = 6 + 1 = 7$
- $prefix[3] = 7 + 3 = 10$
- **Result:**
- `2 6 7 10`

---

## Why Prefix Sums Are Useful

- Suppose we want the sum from index **1 to 3**.
- **Instead of adding again:**
- `4 + 1 + 3`
- **We do:**
- $prefix[3] - prefix[0]$
- **This gives:**
- $10 - 2 = 8$
- So a large sum can be computed **instantly**.



---

## 2️⃣ 2D Prefix Sum (Matrix Version)

- For grids we extend prefix sums to **two dimensions**.
- **Example grid:**

> ```text
> 1 2
> 3 4
> ```

- **Formula:**
- $prefix[r][c] = grid[r][c] + prefix[r-1][c] + prefix[r][c-1] - prefix[r-1][c-1]$
- The subtraction prevents counting the same region twice.

---

## Visual Intuition

- **When computing prefix for a cell, we add:**
- everything above
- everything left
- But the corner region gets counted twice.
- So we subtract it once.



---

## Why This Is Powerful

- **Without prefix sums:**
- Finding sum of a square might require scanning many cells.
- **With prefix sums:**
- We compute the sum **in constant time**.
- This is used in problems like **diamond sums**.

---

## 3️⃣ Parity (Odd and Even)

- **Parity means whether a number is:**
- even
- or
- odd
- **Even numbers:**
- `0 2 4 6 8`
- **Odd numbers:**
- `1 3 5 7 9`

---

## Why Parity Matters in Grid Problems

- Some patterns depend on parity.
- **Example alternating grid:**

> ```text
> 0 1 0
> 1 0 1
> 0 1 0
> ```

- **Notice the rule:**
- cell value changes every step
- **Parity helps detect these patterns. Example formula:**
- $(r + c) \pmod 2$
- This creates a **checkerboard pattern**.



---

## 4️⃣ Toggling Values (Flip 0 ↔ 1)

- Some puzzles involve flipping bits.
- **Example:**
- 0 → 1
- 1 → 0
- This is called **toggling**.

---

## Simple Trick

- **We can toggle using:**
- $1 - \text{value}$
- **Example:**
- $1 - 0 = 1$
- $1 - 1 = 0$
- This trick is used when rows or columns flip.

---

## 5️⃣ Fast Submatrix Calculations

- **Many grid problems ask:**
- What is the sum inside a square or rectangle?
- **Example:**

> ```text
> 1 2 3
> 4 5 6
> 7 8 9
> ```

- **Suppose we want the middle square:**
- `5 6`
- `8 9`
- With prefix sums we compute it instantly using a formula.

---

## 6️⃣ Why These Tools Matter

- **Dynamic Programming often fills a grid. But some problems also require:**
- fast region calculations
- pattern detection
- state changes
- These mathematical tools make those operations efficient.

---

## 🧠 Mental Model

- **Students should understand:**
- **prefix sums** → fast totals
- **parity** → pattern detection
- **toggling** → value flipping
- **submatrix formula** → fast region sums
- These tools combine with **DP tables**.
