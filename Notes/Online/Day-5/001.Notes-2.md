## Tabulation

- **Tabulation** is a bottom-up approach to dynamic programming where we iteratively fill a table (usually an array or a matrix) based on the defined state and transition until we reach the final answer.
- In contrast to memoization, which is a top-down approach, tabulation starts with the simplest subproblems and builds up to the solution of the original problem. 
- Tabulation often requires careful consideration of the order in which subproblems are solved to ensure that all dependencies are computed before they are needed. 

### Tabulation definition

- In tabulation, we create a table (e.g., a 2D array) where each entry corresponds to a subproblem defined by the state. We then fill this table iteratively based on the transition rules until we reach the final answer.
- For example, in the longest increasing path problem, we can create a 2D table `dp` where `dp[i][j]` represents the length of the longest increasing path starting from cell `(i, j)`. We would fill this table iteratively based on the transition rules until we have computed the longest path for all cells, and then we can find the maximum value in the table to get the final answer. 

### Tabulation vs Memoization

- **Memoization** is a top-down approach where we start with the original problem and recursively break it down into subproblems, storing results in a cache to avoid redundant calculations.
- **Tabulation** is a bottom-up approach where we start with the simplest subproblems and iteratively build up to the solution of the original problem by filling a table based on the defined state and transition.
- Both approaches are valid for solving dynamic programming problems, and the choice between them can depend on factors such as the specific problem, the size of the input, and personal preference. Memoization can be easier to implement for some problems, while tabulation can be more efficient in terms of time complexity for others.

### Example problems to understand  tabulation vs memoization

1. **Fibonacci Sequence**
   - Memoization: We can use a recursive approach with a cache to store previously computed Fibonacci numbers.
   - Tabulation: We can create an array where each index represents the Fibonacci number at that position and fill it iteratively from the base cases up to the desired Fibonacci number.
   - Both approaches will yield the same result, but the tabulation method may be more efficient for larger Fibonacci numbers due to reduced overhead from recursive calls.

2. **Longest Increasing Path in a Matrix**
   - Memoization: We can use a recursive approach with a cache to store the longest path starting from each cell.
   - Tabulation: We can create a 2D table where each entry represents the longest path starting from that cell and fill it iteratively based on the transition rules.
   - Both approaches will yield the same result, but the tabulation method may be more efficient for larger matrices due to reduced overhead from recursive calls and better cache locality.
