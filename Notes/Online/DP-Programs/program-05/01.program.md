## Problem Statement

Given a list of coin denominations and their respective quantities, determine the minimum number of coins required to make a given target sum. If it is not possible to make the target sum with the available coins, return -1.
**Twist: This problem is a variation of the classic coin change problem, where each coin denomination has a limited quantity, making it a bounded knapsack problem.**

## Input Format

- The first line contains an integer `m`, the number of different coin denominations.
- The second line contains `m` space-separated integers representing the coin denominations.
- The third line contains `m` space-separated integers representing the quantity of each coin denomination.
- The fourth line contains an integer `target`, the target sum to achieve.

## Output Format

- Output a single integer, the minimum number of coins required to make the target sum. If it is not possible, output `-1`.
- If the target sum is `0`, output `0`.
- If all coin quantities are `0`, output `-1` unless the target is `0`, in which case output `0`.
- If the target sum is negative, output `-1`.
- If the target sum is less than the smallest coin denomination, output `-1`.
- If the target sum is exactly equal to one of the coin denominations and the quantity of that denomination is greater than `0`, output `1`.

## Constraints

- `1 <= m <= 100`
- `1 <= coin denominations[i] <= 10^4`
- `0 <= quantity[i] <= 1000`
- `0 <= target <= 10^5`

## Sample Input

```int
3
1 2 3
2 1 1
5
```

## Sample Output

```int
2
```

## Explanation

To make the target sum of `5`, we can use one `2` coin and one `3` coin, which gives us a total of `2` coins. Alternatively, we could use two `1` coins and one `3` coin, but that would require `3` coins in total. Therefore, the minimum number of coins required is `2`.

Therefore, the output is `-1`.

## Knapsack DP 

**Definition:** The knapsack problem is a combinatorial optimization problem that asks for the maximum value that can be put in a knapsack of a given capacity. In the context of the coin change problem with limited coins, we can think of the target sum as the capacity of the knapsack and the coins as items with certain values (denominations) and weights (quantities).

**Eli5 Explanation:** Imagine you have a backpack (knapsack) that can hold a certain weight (target sum). You have different items (coins) that you can put in the backpack, but each item has a limited quantity. The goal is to fill the backpack with the least number of items while reaching the target weight. This is similar to trying to make a certain amount of money using a limited number of coins.

To solve this problem, we can use a dynamic programming approach similar to the knapsack problem. We will create a 2D DP array where `dp[i][j]` represents the minimum number of coins needed to make the sum `j` using the first `i` types of coins. We will iterate through each coin denomination and its quantity, updating our DP array accordingly.

## Trace run for the sample input:

Let's trace the DP array for the sample input:

- Coin denominations: [1, 2, 3]
- Quantities: [2, 1, 1]
- Target: 5
- DP array initialization:
  - `dp[0][0] = 0` (0 coins needed to make sum 0)
  - `dp[0][j] = inf` for `j > 0` (impossible to make positive sums with 0 coins)
  - `dp[i][0] = 0` for all `i` (0 coins needed to make sum 0)
  - `dp[i][j] = inf` for `j > 0` and `i > 0` (initially set to infinity)
- Processing coin 1 (quantity 2):
  - For sum 1: `dp[1][1] = min(dp[0][1], dp[1][0] + 1) = min(inf, 0 + 1) = 1`
  - For sum 2: `dp[1][2] = min(dp[0][2], dp[1][1] + 1) = min(inf, 1 + 1) = 2`
  - For sum 3: `dp[1][3] = min(dp[0][3], dp[1][2] + 1) = min(inf, 2 + 1) = 3`
  - For sum 4: `dp[1][4] = min(dp[0][4], dp[1][3] + 1) = min(inf, 3 + 1) = 4`
  - For sum 5: `dp[1][5] = min(dp[0][5], dp[1][4] + 1) = min(inf, 4 + 1) = 5`
- Processing coin 2 (quantity 1):
  - For sum 2: `dp[2][2] = min(dp[1][2], dp[2][0] + 1) = min(2, 0 + 1) = 1`
  - For sum 3: `dp[2][3] = min(dp[1][3], dp[2][1] + 1) = min(3, inf) = 3`
  - For sum 4: `dp[2][4] = min(dp[1][4], dp[2][2] + 1) = min(4, 1 + 1) = 2`
  - For sum 5: `dp[2][5] = min(dp[1][5], dp[2][3] + 1) = min(5, 3 + 1) = 3`
- Processing coin 3 (quantity 1):
  - For sum 3: `dp[3][3] = min(dp[2][3], dp[3][0] + 1) = min(3, 0 + 1) = 1`
  - For sum 4: `dp[3][4] = min(dp[2][4], dp[3][1] + 1) = min(2, inf) = 2`
  - For sum 5: `dp[3][5] = min(dp[2][5], dp[3][2] + 1) = min(3, inf) = 3`

After processing all coins, the minimum number of coins needed to make the target sum of `5` is found in `dp[3][5]`, which is `2`. This corresponds to using one `2` coin and one `3` coin.

## cpp solution

```cpp

#include <iostream>
#include <vector>
#include <algorithm>
#include <climits>

using namespace std;

int coinChangeWithLimitedCoins(vector<int>& denominations, vector<int>& quantities, int target) {
    int m = denominations.size();
    vector<vector<int>> dp(m + 1, vector<int>(target + 1, INT_MAX));
    
    // Base case: 0 coins needed to make sum 0
    for (int i = 0; i <= m; i++) {
        dp[i][0] = 0;
    }
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= target; j++) {
            dp[i][j] = dp[i - 1][j]; // Not using the current coin
            for (int k = 1; k <= quantities[i - 1] && k * denominations[i - 1] <= j; k++) {
                if (dp[i - 1][j - k * denominations[i - 1]] != INT_MAX) {
                    dp[i][j] = min(dp[i][j], dp[i - 1][j - k * denominations[i - 1]] + k);
                }
            }
        }
    }
    
    return dp[m][target] == INT_MAX ? -1 : dp[m][target];
}

int main() {
    int m;
    cin >> m;
    vector<int> denominations(m);
    vector<int> quantities(m);
    
    for (int i = 0; i < m; i++) {
        cin >> denominations[i];
    }
    
    for (int i = 0; i < m; i++) {
        cin >> quantities[i];
    }
    
    int target;
    cin >> target;
    
    int result = coinChangeWithLimitedCoins(denominations, quantities, target);
    cout << result << endl;
    
    return 0;
}
```
