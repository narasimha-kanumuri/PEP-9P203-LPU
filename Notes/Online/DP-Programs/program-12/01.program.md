# Program 12: Knapsack with Item Dependencies

## Knapsack Problem with Item Dependencies

---

## Problem

You are given a list of items, where each item has a weight, a value, and a dependency on another item. You can only take an item if you also take its dependency. Your task is to determine the maximum value you can achieve with a knapsack of a given capacity. The dependencies form a directed acyclic graph (DAG), meaning there are no circular dependencies. You need to consider the dependencies while deciding which items to include in the knapsack.

---

## Constraints

- 1 ≤ number of items ≤ 100
- 1 ≤ weight of each item ≤ 1000
- 1 ≤ value of each item ≤ 1000
- 1 ≤ knapsack capacity ≤ 1000
- Dependencies are given as pairs (item, dependency), where item and dependency are indices of the items in the list.
- The answer may be large, return it modulo 10^9 + 7

---

## Example

### Input

Items: [(weight=2, value=3, dependency=-1), (weight=3, value=4, dependency=0), (weight=4, value=5, dependency=1)]
Knapsack capacity: 5

### Output

Maximum value: 7

## ELI5 Explanation of above test case

In this example, we have three items. The first item has a weight of 2 and a value of 3, and it has no dependencies (dependency = -1). The second item has a weight of 3 and a value of 4, but it depends on the first item (dependency = 0). The third item has a weight of 4 and a value of 5, but it depends on the second item (dependency = 1). With a knapsack capacity of 5, we can only take the first two items because taking the third item would exceed the capacity. The total weight of the first two items is 2 + 3 = 5, which fits within the knapsack capacity, and their total value is 3 + 4 = 7. Therefore, the maximum value we can achieve is 7.

## Implementation Narrative (English → Code)

1. We will use dynamic programming to solve this problem. We will create a 2D array `dp` where `dp[i][w]` represents the maximum value that can be achieved with the first `i` items and a knapsack capacity of `w`.
2. We will also create a mapping of item dependencies to easily access the dependencies for each item.
3. We will iterate through each item and for each item, we will check if we can include it in the knapsack. To include an item, we need to ensure that we also include its dependency (if it has one).
4. If we can include the item, we will update our `dp` array by considering the value of including the item and its dependency, as well as the value of not including the item.
5. We will take all calculations modulo 10^9 + 7 to handle large numbers.
6. Finally, the maximum value we can achieve with the given knapsack capacity will be found in `dp[number_of_items][knapsack_capacity]`, which we will return as the final answer.

```cpp

#include <iostream>
#include <vector>
#include <unordered_map>
using namespace std;

const int MOD = 1e9 + 7;
int knapsackWithDependencies(vector<pair<int, int>>& items, vector<int>& dependencies, int capacity) {
    int n = items.size();
    vector<vector<int>> dp(n + 1, vector<int>(capacity + 1, 0));
    unordered_map<int, vector<int>> dependencyMap;

    // Build the dependency map
    for (int i = 0; i < n; i++) {
        if (dependencies[i] != -1) {
            dependencyMap[dependencies[i]].push_back(i);
        }
    }

    for (int i = 1; i <= n; i++) {
        int weight = items[i - 1].first;
        int value = items[i - 1].second;
        int dep = dependencies[i - 1];

        for (int w = 0; w <= capacity; w++) {
            // Case when we do not take the item
            dp[i][w] = dp[i - 1][w];

            // Case when we take the item
            if (dep == -1 || dp[dep + 1][w - weight] > 0) { // Check if dependency is satisfied
                if (w >= weight) {
                    dp[i][w] = max(dp[i][w], (dp[dep + 1][w - weight] + value) % MOD);
                }
            }
        }
    }

    return dp[n][capacity];
}

int main() {
    vector<pair<int, int>> items = {{2, 3}, {3, 4}, {4, 5}};
    vector<int> dependencies = {-1, 0, 1};
    int capacity = 5;
    cout << "Maximum value: " << knapsackWithDependencies(items, dependencies, capacity) << endl;
    return 0;
}
```
