# Problem: Grid Paths with One Obstacle Removal

## concepts used

Dynamic Programming, State Augmentation, Grid Traversal, Combinatorics, Obstacle Removal

## Problem Statement

You are given a grid of size `m x n` where each cell is either empty (0) or contains an obstacle (1). You need to find the number of unique paths from the top-left corner of the grid (0, 0) to the bottom-right corner (m-1, n-1). You can only move either down or right at any point in time. However, you are allowed to remove exactly one obstacle from the grid to help you reach the destination.

**Twist:** To solve this problem, you can use a dynamic programming approach with state augmentation. You can define a 3D DP array `dp[r][c][used]` where `r` and `c` represent the current row and column, and `used` is a binary state indicating whether you have already removed an obstacle (0 for not used, 1 for used).

## Input Format

- The first line contains two integers `m` and `n`, representing the number of rows and columns in the grid.
- The next `m` lines each contain `n` integers (0 or 1), representing the grid.
- 0 indicates an empty cell, and 1 indicates an obstacle.
- The top-left corner (0, 0) and the bottom-right corner (m-1, n-1) will always be empty (0).
- You can assume that there is at least one valid path from the top-left to the bottom-right corner.
  
## Output Format

- Output a single integer, the number of unique paths from the top-left corner to the bottom-right corner, considering that you can remove exactly one obstacle.
- Since the number of paths can be large,
- If there are no valid paths, return 0.
- If the grid is empty (m = 0 or n = 0), return 0.
- If the top-left corner (0, 0) or the bottom-right corner (m-1, n-1) contains an obstacle, return 0.
- If there are no obstacles in the grid, return the number of unique paths without any removals.
- If there is only one obstacle in the grid, return the number of unique paths after removing that obstacle.
- If there are multiple obstacles, consider all possible removals and sum the valid paths for each case.
- If the grid is such that even after removing one obstacle, there are no valid paths, return 0.
- If the grid is large, ensure that your solution is optimized to run within reasonable time limits.

## Constraints

- `1 <= m, n <= 100`
- `grid[i][j]` is either 0 or 1.
- The top-left corner (0, 0) and the bottom-right corner (m-1, n-1) will always be 0 (empty).
- Return the result modulo `10^9 + 7`.

## Complete algorithm in plain English

*Note:Also added why for each step*

1. **Initialize a 3D DP array**: Create a 3D DP array `dp[m][n][2]` to store the number of paths to reach each cell with and without using the obstacle removal. This allows us to keep track of both states (used and not used) for each cell.
2. **Base case**: Set `dp[0][0][0] = 1` since there is one way to be at the starting point without removing any obstacle. This serves as the starting point for our DP calculations.
3. **Fill the DP array**: Iterate through each cell in the grid. For each cell, check if it is an obstacle or not. If it is an obstacle and we haven't used our removal yet, we can remove it and add paths from the top and left cells. If it's not an obstacle, we simply add paths from the top and left cells based on whether we've used our removal or not. This step builds our solution iteratively based on previously computed results.
4. **Calculate the final result**: The total number of paths to reach the bottom-right corner will be the sum of paths with no obstacle removed and paths with one obstacle removed, i.e., `dp[m-1][n-1][0] + dp[m-1][n-1][1]`. This gives us the total unique paths considering both scenarios.
5. **Return the result modulo `10^9 + 7`**: Since the number of paths can be large, we take the result modulo `10^9 + 7` to prevent integer overflow and ensure that our output is manageable.
6. **Handle edge cases**: Ensure that if there are no valid paths, or if the grid is empty, or if the start or end points are obstacles, we return 0. This ensures that our solution is robust and handles all possible input scenarios correctly.
7. **Optimize for large grids**: The DP approach ensures that we can compute the number of paths efficiently even for larger grids, as we are building our solution iteratively and avoiding redundant calculations.

## Cpp implementation

```cpp

#include <iostream>
#include <vector>
using namespace std;

const int MOD = 1e9 + 7;

int uniquePathsWithObstacles(vector<vector<int>>& grid) {
    int m = grid.size();
    int n = grid[0].size();
    
    // dp[r][c][used] - number of paths to reach (r, c) with 'used' indicating if an obstacle has been removed
    vector<vector<vector<int>>> dp(m, vector<vector<int>>(n, vector<int>(2, 0)));
    
    // Base case: starting point
    dp[0][0][0] = 1; // No obstacle removed at the start
    
    for (int r = 0; r < m; ++r) {
        for (int c = 0; c < n; ++c) {
            if (r == 0 && c == 0) continue; // Skip the starting point
            
            for (int used = 0; used <= 1; ++used) {
                if (grid[r][c] == 1) { // If there's an obstacle
                    if (used == 0) { // We can remove this obstacle
                        dp[r][c][1] = (dp[r][c][1] + dp[r-1][c][0]) % MOD; // From top
                        dp[r][c][1] = (dp[r][c][1] + dp[r][c-1][0]) % MOD; // From left
                    }
                } else { // No obstacle
                    dp[r][c][used] = (dp[r][c][used] + dp[r-1][c][used]) % MOD; // From top
                    dp[r][c][used] = (dp[r][c][used] + dp[r][c-1][used]) % MOD; // From left
                }
            }
        }
    }
    
    return (dp[m-1][n-1][0] + dp[m-1][n-1][1]) % MOD;
}

int main() {
    int m, n;
    cin >> m >> n;
    vector<vector<int>> grid(m, vector<int>(n));
    
    for (int i = 0; i < m; ++i) {
        for (int j = 0; j < n; ++j) {
            cin >> grid[i][j];
        }
    }
    
    cout << uniquePathsWithObstacles(grid) << endl;
    return 0;
}
```
