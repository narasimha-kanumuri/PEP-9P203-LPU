# Program 11: Unique Paths with Energy Boost

## Count Unique Paths in a Grid with an Energy Boost

---

## Problem

You are given a grid of size m x n. You start at the top-left corner (0, 0) and want to reach the bottom-right corner (m-1, n-1). You can only move either down or right at any point in time.
However, you have one energy boost that allows you to skip one cell in your path. This means you can move from (i, j) to (i+2, j) or (i, j+2) once during your journey.
Calculate the total number of unique paths from the top-left corner to the bottom-right corner, considering the energy boost.

---

## Constraints

- 1 ≤ m, n ≤ 100
- You can only use the energy boost once
- You cannot skip outside the grid boundaries
- The answer may be large, return it modulo 10^9 + 7
  
---

## Example

### Input

m = 3, n = 3

### Output

Number of unique paths: 12

---

## ELI5 Explanation of above test case

In a 3x3 grid, you can move right or down to reach the bottom-right corner. Normally, there are 6 unique paths without using the energy boost. However, with the energy boost, you can skip one cell, which allows for additional paths. For example, you can skip from (0, 0) to (2, 0) or from (0, 0) to (0, 2), and then continue moving right or down. This results in a total of 12 unique paths when considering the energy boost.

---

## Implementation Narrative (English → Code)

1. We can use dynamic programming to solve this problem. We will create a 2D array `dp` where `dp[i][j]` represents the number of unique paths to reach cell (i, j) without using the energy boost.
2. We will also create another 2D array `boostDp` where `boostDp[i][j]` represents the number of unique paths to reach cell (i, j) using the energy boost.
3. We will fill the `dp` array using the standard unique paths logic: `dp[i][j] = dp[i-1][j] + dp[i][j-1]`.
4. For the `boostDp` array, we will consider the cases where we use the energy boost:
   - If we skip down from (i-2, j), then `boostDp[i][j] += dp[i-2][j]` (if i-2 >= 0).
   - If we skip right from (i, j-2), then `boostDp[i][j] += dp[i][j-2]` (if j-2 >= 0).
   - Additionally, we need to consider the paths where the boost has already been used: `boostDp[i][j] += boostDp[i-1][j] + boostDp[i][j-1]`.
   - We will take all calculations modulo 10^9 + 7 to handle large numbers.
   - Finally, the total number of unique paths to reach (m-1, n-1) will be the sum of `dp[m-1][n-1]` and `boostDp[m-1][n-1]`.
   - We will return this total as the final answer.

```cpp

#include <vector>
using namespace std;

class Solution {
public:
    int uniquePathsWithEnergyBoost(int m, int n) {
        const int MOD = 1e9 + 7;
        vector<vector<int>> dp(m, vector<int>(n, 0));
        vector<vector<int>> boostDp(m, vector<int>(n, 0));

        // Base case: there's one way to reach the starting cell
        dp[0][0] = 1;

        for (int i = 0; i < m; ++i) {
            for (int j = 0; j < n; ++j) {
                if (i == 0 && j == 0) continue; // Skip the starting cell

                // Calculate paths without using energy boost
                if (i > 0) dp[i][j] = (dp[i][j] + dp[i-1][j]) % MOD;
                if (j > 0) dp[i][j] = (dp[i][j] + dp[i][j-1]) % MOD;

                // Calculate paths using energy boost
                if (i > 1) boostDp[i][j] = (boostDp[i][j] + dp[i-2][j]) % MOD;
                if (j > 1) boostDp[i][j] = (boostDp[i][j] + dp[i][j-2]) % MOD;
                if (i > 0) boostDp[i][j] = (boostDp[i][j] + boostDp[i-1][j]) % MOD;
                if (j > 0) boostDp[i][j] = (boostDp[i][j] + boostDp[i][j-1]) % MOD;
            }
        }

        return (dp[m-1][n-1] + boostDp[m-1][n-1]) % MOD;
    }
};

//main

#include <iostream>

int main() {
    Solution sol;
    int m = 3, n = 3;
    cout << "Number of unique paths: " << sol.uniquePathsWithEnergyBoost(m, n) << endl;
    return 0;
}
```
