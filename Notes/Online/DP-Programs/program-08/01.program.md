## Problem Statement

Given a 2D grid of integers, find the maximum sum of a diamond-shaped region of size K. A diamond-shaped region of size K consists of cells that are at most K-1 steps away from a central cell in terms of Manhattan distance. However, if any cell within the diamond-shaped region contains the value -1, it breaks the shape and the sum for that region is considered invalid. Your task is to compute the maximum valid diamond sum in the grid.
**Twist: This problem can be solved using a combination of 2D prefix sums for efficient area sum calculation, along with dynamic programming to validate the shape and handle the blocked cells.**

## Input Format

- The first line contains three integers `n`, `m`, and `K`, representing the number of rows, columns, and the size of the diamond, respectively.
- The next `n` lines each contain `m` integers, representing the values in the grid. The values can be positive integers or -1, where -1 indicates a blocked cell.
- The grid is guaranteed to have at least one valid diamond-shaped region of size K.

## Output Format

- Output a single integer, the maximum sum of a valid diamond-shaped region of size K in the grid.
- If there are no valid diamond-shaped regions, output 0.
- The sum of the diamond-shaped region is calculated by summing the values of all cells that are part of the diamond, excluding any cells with a value of -1. If any cell in the diamond has a value of -1, the entire diamond is considered invalid and should not contribute to the maximum sum.
- The diamond-shaped region is defined as all cells (i, j) such that the Manhattan distance from the central cell (x, y) is less than K, i.e., |i - x| + |j - y| < K.
- The central cell of the diamond can be any cell in the grid, and the diamond can extend in all four directions (up, down, left, right) as long as the Manhattan distance condition is satisfied and there are no blocked cells within the diamond.

```grid

```

## Sample Output

```int
19
```
