## Problem Statement

Given a 2D grid of integers, find the maximum sum of a diamond-shaped region of size K. A diamond-shaped region of size K consists of cells that are at most K-1 steps away from a central cell in terms of Manhattan distance. However, if any cell within the diamond-shaped region contains the value -1, it breaks the shape and the sum for that region is considered invalid. Your task is to compute the maximum valid diamond sum in the grid.
**Twist: This problem can be solved using a combination of 2D prefix sums for efficient area sum calculation, along with dynamic programming to validate the shape and handle the blocked cells.**

## Input Format

- The first line contains three integers `n`, `m`, and `K`, representing the number of rows, columns, and the size of the diamond, respectively.
- The next `n` lines each contain `m` integers, representing the values in the grid. The values can be positive integers or -1, where -1 indicates a blocked cell.
- The grid is guaranteed to have at least one valid diamond-shaped region of size K.

## Output Format

- Output a single integer, the maximum sum of a valid diamond-shaped region of size K in the grid.
- If there are no valid diamond-shaped regions, output 0.
- The sum of the diamond-shaped region is calculated by summing the values of all cells that are part of the diamond, excluding any cells with a value of -1. If any cell in the diamond has a value of -1, the entire diamond is considered invalid and should not contribute to the maximum sum.
- The diamond-shaped region is defined as all cells (i, j) such that the Manhattan distance from the central cell (x, y) is less than K, i.e., |i - x| + |j - y| < K.
- The central cell of the diamond can be any cell in the grid, and the diamond can extend in all four directions (up, down, left, right) as long as the Manhattan distance condition is satisfied and there are no blocked cells within the diamond.

## Constraints

- 1 ≤ n, m ≤ 100
- 1 ≤ K ≤ min(n, m)
- -1 ≤ grid[i][j] ≤ 1000
- The grid will contain at least one valid diamond-shaped region of size K.
- The maximum sum of a valid diamond-shaped region will fit within the range of a 32-bit signed integer.
- The input will be provided in a format that allows for efficient reading and processing, and the output should be produced in a timely manner given the constraints.
- The solution should be optimized to handle the worst-case scenario within the given constraints, ensuring that it runs efficiently for larger grid sizes and diamond sizes.

## Step by step English to Algorithm of the problem

1. **Input Reading**: Read the dimensions of the grid (n, m) and the size of the diamond (K). Then read the grid values into a 2D array.
2. **Precompute Prefix Sums**: Create a 2D prefix sum array to allow for efficient calculation of the sum of any rectangular region in the grid. This will help in quickly calculating the sum of the diamond-shaped regions.
3. **Iterate Over Potential Centers**: Loop through each cell in the grid to consider it as the potential center of a diamond-shaped region.
4. **Validate Diamond Shape**: For each potential center, check the cells that would be included in the diamond-shaped region based on the Manhattan distance condition. If any of these cells contain -1, mark the diamond as invalid and skip to the next center.
5. **Calculate Diamond Sum**: If the diamond is valid, use the prefix sum array to calculate the sum of the cells in the diamond-shaped region efficiently.
6. **Track Maximum Sum**: Keep track of the maximum valid diamond sum encountered during the iterations.
7. **Output the Result**: After iterating through all potential centers, output the maximum valid diamond sum. If no valid diamonds were found, output 0.

## Cpp 

```cpp

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    int n, m, K;
    cin >> n >> m >> K;
    
    vector<vector<int>> grid(n, vector<int>(m));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            cin >> grid[i][j];
        }
    }
    
    // Step 2: Precompute prefix sums for efficient area sum calculation by using 1-based indexing
    vector<vector<int>> prefixSum(n + 1, vector<int>(m + 1, 0));// 1-based indexing for easier calculations
    for (int i = 1; i <= n; i++) { // Fill prefix sum array
        for (int j = 1; j <= m; j++) { // Calculate prefix sum
            prefixSum[i][j] = grid[i - 1][j - 1] + prefixSum[i - 1][j] + prefixSum[i][j - 1] - prefixSum[i - 1][j - 1];// Prefix sum calculation
        }
    }
    
    int maxDiamondSum = 0;
    
    // Step 3: Iterate over potential centers
    for (int x = 0; x < n; x++) {
        for (int y = 0; y < m; y++) {
            bool isValidDiamond = true;
            int diamondSum = 0;
            
            // Step 4: Validate diamond shape and calculate sum
            for (int dx = -K + 1; dx < K; dx++) {
                for (int dy = -K + 1; dy < K; dy++) {
                    if (abs(dx) + abs(dy) < K) { // Check if within diamond shape
                        int nx = x + dx;
                        int ny = y + dy;
                        
                        if (nx >= 0 && nx < n && ny >= 0 && ny < m) { // Check bounds
                            if (grid[nx][ny] == -1) { // Check for blocked cell
                                isValidDiamond = false;
                                break;
                            } else {
                                diamondSum += grid[nx][ny]; // Add to diamond sum
                            }
                        }
                    }
                }
                if (!isValidDiamond) break;
            }
            
            // Step 6: Track maximum sum
            if (isValidDiamond) {
                maxDiamondSum = max(maxDiamondSum, diamondSum);
            }
        }
    }
    
    // Step 7: Output the result
    cout << maxDiamondSum << endl;

    return 0;
}
```
