# 🏠 Program 2: The House Robber (Non-Adjacent Max Sum)

---

## 🔖 Concepts Involved

- Dynamic Programming (DP), Overlapping Subproblems, Local vs. Global Optimization, State Transition

---

## 📖 Problem Story / Context

- You are a professional robber planning to rob houses along a street. 
- Each house has a certain amount of money stashed.

- **The Constraint:**
- All houses at this place are arranged in a straight line. 
- Adjacent houses have security systems connected; it will automatically contact the police if **two adjacent houses** are broken into on the same night.

- **Goal:**
- Determine the maximum amount of money you can rob tonight **without alerting the police**.

---

## 🧠 Core Insight (ELI5)

- At every house $i$, you face a binary choice:

- **1. Rob house $i$:**
- You gain `nums[i]`, but you must skip the previous house ($i-1$). Your total is `nums[i] + dp[i-2]`.

- **2. Skip house $i$:**
- You keep the best loot found up to the previous house. Your total is `dp[i-1]`.

- **Decision:**
- Choose the maximum of these two options.

---

## ⚙️ State & Transition

**State Definition:**
- `dp[i]` = Maximum money earned from the first $i$ houses.

**Transition Formula:**
- $$dp[i] = \max(dp[i-1], dp[i-2] + nums[i])$$

**Base Cases:**
- `dp[0] = nums[0]`
- `dp[1] = max(nums[0], nums[1])`

---

## 🔍 Sample Logic Trace

- **Input:** `nums = [2, 7, 9, 3, 1]`

| House (i) | Value | Decision: $\max(dp[i-1], dp[i-2] + val)$ | `dp[i]` |
| :--- | :--- | :--- | :--- |
| 0 | 2 | Base Case | 2 |
| 1 | 7 | $\max(2, 7)$ | 7 |
| 2 | 9 | $\max(7, 2 + 9)$ | 11 |
| 3 | 3 | $\max(11, 7 + 3)$ | 11 |
| 4 | 1 | $\max(11, 11 + 1)$ | 12 |

- **Result:** 12



---

## 💻 Optimal Implementation (C++)

```cpp
int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    if (n == 1) return nums[0];

    // Space Optimized DP
    int prev2 = nums[0];
    int prev1 = max(nums[0], nums[1]);

    for (int i = 2; i < n; i++) {
        int curr = max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }

    return prev1;
}
```

---

## ⏱ Complexity Analysis

- **Time Complexity:**
- `O(N)`
- → We iterate through the list of houses exactly once.

- **Space Complexity:**
- `O(1)`
- → By using only two variables (`prev1`, `prev2`), we avoid an $O(N)$ array.

---

## 🎯 Core Pattern: "Pick vs. Skip"

- This problem is the foundation for:
- **Weighted Job Scheduling**
- **Maximum Weight Independent Set**
- **DP on Sequences**

- **Mental Model:**
- If a decision at step $i$ invalidates step $i-1$, you are likely looking at a House Robber variation.

---

## 🚧 Common Pitfalls

- **Greedy Failure:**
- Thinking "just pick the largest numbers" fails for `[2, 1, 1, 2]`. Greedy picks `2 + 2 = 4`, but if the middle were higher, greedy often misses the global optimum.
- **Index Out of Bounds:**
- Forgetting to handle cases where $N < 2$ before accessing `nums[1]` or `prev2`.
- **Recursion Depth:**
- Using raw recursion without memoization leads to `O(2^n)` complexity.
