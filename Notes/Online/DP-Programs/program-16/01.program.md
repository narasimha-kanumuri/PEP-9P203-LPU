# 15️⃣ Minimum Path Sum with One Cell Skip

## Problem Statement

Given a 2D grid of non-negative integers, find the minimum path sum from the top-left corner to the bottom-right corner of the grid. You can only move either down or right at any point in time. However, you are allowed to skip one cell during your path, which means you can move from one cell to another cell that is not adjacent (i.e., you can skip one cell in between).

## Constraints

- `m == grid.length`
- `n == grid[i].length`
- `1 <= m, n <= 100`
- `0 <= grid[i][j] <= 100`


## Examples

### Example 1

**Input**: grid = [[1,3,1],[1,5,1],[4,2,1]]
**Output**: 7

**Explanation**: The path with the minimum sum is 1 → 3 → 1 → 1 → 1, which has a sum of 7. You can skip the cell with value 5.

### Example 2

**Input**: grid = [[1,2,3],[4,5,6]]
**Output**: 12

**Explanation**: The path with the minimum sum is 1 → 2 → 3 → 6, which has a sum of 12. You can skip the cell with value 5.

### Example 3

**Input**: grid = [[1,2],[1,1]]
**Output**: 2

**Explanation**: The path with the minimum sum is 1 → 1, which has a sum of 2. You can skip the cell with value 2.

## ELI5 Explanation

Imagine you have a grid of numbers, and you want to find the path from the top-left corner to the bottom-right corner that has the smallest sum of numbers. You can only move right or down, but you have a special power that allows you to skip one cell along the way. This means you can jump over one cell and move to the next one, which might help you get a smaller sum.

## Implementation Narrative

To solve this problem, we can use dynamic programming to keep track of the minimum path sum at each cell. We will create a 2D array `dp` where `dp[i][j]` represents the minimum path sum to reach cell `(i, j)` from the top-left corner.

1. We will initialize the base case where `dp[0][0] = grid[0][0]`, since the minimum path sum to reach the starting cell is just the value of that cell.
2. We will fill the first row and first column of the `dp` table, since the only way to reach those cells is by moving right or down respectively.
3. For the rest of the cells, we will calculate the minimum path sum by taking the minimum of the two possible moves (right or down) and adding the value of the current cell.
4. To account for the one cell skip, we will iterate through the `dp` table and for each cell, we will check if skipping the next cell (either right or down) results in a smaller path sum. We will update the `dp` table accordingly.
5. Finally, the value at `dp[m-1][n-1]` will give us the minimum path sum from the top-left corner to the bottom-right corner with at most one cell skip.
6. We will also need to handle edge cases where skipping a cell might not be possible (e.g., when we are at the last row or last column).
