# 🟢 LEVEL 10 — Advanced Dynamic Programming Concepts

---

## Concepts

- So far we learned basic DP like $dp[i]$ or $dp[r][c]$.
- These store **one piece of information per state**.
- But some problems require **more information**.
- That is where advanced DP ideas appear, helping us solve the hardest "twisted" problems.

---

## 1️⃣ State Augmentation (Adding Extra Information)

- Sometimes a problem depends on **more than just position**.
- **Example situation:**
- "You can teleport once."
- **Now reaching a cell depends on:**
- position (row, col)
- **AND** whether teleport was already used.
- One state is not enough to describe this situation.

---

## Example State

- **Instead of:**
- $dp[r][c]$
- **We store:**
- $dp[r][c][t]$
- **Where:**
- $t = 0 \rightarrow$ teleport unused
- $t = 1 \rightarrow$ teleport used
- This means each cell has **two possible states** depending on your history.



---

## 2️⃣ Multi-Dimensional DP

- **Each dimension represents a factor affecting the answer.**
- **Example Scenario:**
- Suppose a robot moves in a grid but has **3 energy levels**.
- **State becomes:**
- $dp[row][col][energy]$
- Now the answer depends on your exact position plus how much fuel you have left.

---

## 3️⃣ State Compression

- **State compression reduces memory usage.**
- **Idea:**
- Instead of storing all rows in a giant $dp[r][c]$ table, we store only the **previous row**.
- **Example:**
- `currentRow[c]`
- `previousRow[c]`
- Because each cell only depends on the one directly above or to the left.

---

## 4️⃣ Transition Optimization

- **Sometimes transitions are slow (e.g., $O(n^2)$).**
- Optimization techniques reduce them to $O(n \log n)$ or $O(n)$.
- **Methods include:**
- prefix sum optimization
- monotonic queues
- segment trees
- These appear when a cell depends on a whole range of previous cells rather than just one.

---

## 5️⃣ DP on Graphs

- **Some DP problems can be viewed as graphs.**
- **Example grid:**
- cells = nodes
- movements = edges
- **DP becomes:**
- Finding the shortest path, longest path, or counting all possible paths.
- Each node stores a DP value calculated from its neighbors.



---

## 6️⃣ Order of Computation (Topological Order)

- **DP requires solving smaller problems first.**
- In a grid, this usually means moving:
- $(0,0) \rightarrow (0,1) \rightarrow (0,2)$
- $\downarrow$
- $(1,0) \rightarrow (1,1) \rightarrow (1,2)$
- This ensures that when you calculate a cell, the cells it depends on are **already finished**.

---

## 7️⃣ Space-Time Tradeoffs

- **DP often trades memory for speed.**
- **Without DP:** Time is exponential (slow), memory is small.
- **With DP:** Time is polynomial (fast), memory is larger.
- We use **state compression** (Level 3) to keep that memory usage as low as possible.

---

## 8️⃣ DP Pattern Recognition

- **Common DP patterns to look for:**
- 1D DP (Linear)
- 2D Grid DP (Matrices)
- Knapsack DP (Choices)
- Interval DP (Ranges)
- Bitmask DP (Subsets)

---

## 9️⃣ DP Debugging Mindset

- **Always ask these four questions:**
- 1️⃣ What does each state represent?
- 2️⃣ What previous states affect it?
- 3️⃣ What are the base cases (starting points)?
- 4️⃣ In what order should we compute?

---

## 🔟 Mental Model for Advanced DP

- **Imagine DP as building a solution layer by layer:**
- Huge problem
- $\downarrow$ Break into smaller states
- $\downarrow$ Store answers
- $\downarrow$ Reuse answers
- $\downarrow$ Build final solution
