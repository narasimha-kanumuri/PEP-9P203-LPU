# 🟢 LEVEL 7 — Grid Dynamic Programming Concepts

- **Grid DP** is simply:

- **Dynamic Programming applied to a grid (matrix):**

- Instead of storing answers in:

- `dp[i]`

- we store answers in:

- `dp[row][column]`

- Each cell of the grid becomes a small problem.

---

## 1️⃣ The Core Idea

- In grid DP:

- **Each cell’s answer depends on answers of nearby cells.:**

- **Example grid:**

```text
1 2 3
4 5 6
7 8 9
```

- Suppose we want the maximum sum path from top-left to each cell.

- We store answers like this:

- **dp table:**

- **Example:**

```text
1  3  6
5 10 16
12 20 29
```

- Each number is the best result up to that cell.



---

## 2️⃣ What is the DP State?

- The state describes what each cell stores.

- **Example state:**

- `dp[r][c]` = best path sum reaching cell $(r,c)$

- **Meaning:**
- The value tells us the best result if we end at this cell.

- Every DP problem starts by defining the state.

---

## 3️⃣ Cell Dependencies

- Each cell usually depends on previous cells.

- **Example movement rules:**

- You can move:
- → right
- ↓ down

- So cell $(r,c)$ depends on:

- **top cell → (r-1,c):**
- **left cell → (r,c-1):**

- **Transition rule:**

$$dp[r][c] = grid[r][c] + \max(dp[r-1][c], dp[r][c-1])$$

- **Meaning:**
- The best path to $(r,c)$ comes from the best neighbor.

---

## 4️⃣ Base Cells

- DP must start somewhere.

- **Example start cell:**

- **(0,0):**

- **Base case:**

- $dp[0][0] = grid[0][0]$

- This is the first solved subproblem.

- Then we fill the grid gradually.

---

## 5️⃣ DP Traversal Order

- Cells must be computed only after their dependencies are known.

- **Example:**

- **Fill row by row:**
- → → →
- → → →
- → → →

- **Why?**
- Because each cell depends on:

- **top:**
- **left:**

- Those cells must already be computed.

---

## 6️⃣ Grid as a Dependency Graph

- A grid DP can be visualized as a directed graph.

- **Example dependencies:**

```text
(0,0) → (0,1) → (0,2)
   ↓       ↓       ↓
(1,0) → (1,1) → (1,2)
   ↓       ↓       ↓
(2,0) → (2,1) → (2,2)
```

- Arrows show which cells depend on which.

- This graph has no cycles.

- That’s why DP works.



---

## 7️⃣ Types of Grid DP Problems

- Different grid problems use different transitions.

- Common types include:

- **Path Counting:**

- **Example question:**
- How many ways to reach bottom-right?

- **Transition:**
- $dp[r][c] = dp[r-1][c] + dp[r][c-1]$

- **Meaning:**
- Paths come from top or left.

- **Maximum / Minimum Path:**

- **Example:**
- Find best path sum.

- **Transition:**
- $dp[r][c] = grid[r][c] + \max(dp[r-1][c], dp[r][c-1])$

- **Longest Pattern:**

- **Example:**
- largest square
- largest alternating pattern

- Transition depends on multiple neighbors.

---

## 8️⃣ Multi-State Grid DP

- Sometimes each cell needs multiple states.

- **Example from your puzzles:**

- **teleport allowed once:**

- **State becomes:**

- `dp[r][c][t]`

- **Where:**

- **t = 0 → teleport unused:**
- **t = 1 → teleport used:**

- This means the same cell can have two different answers.

---

## 9️⃣ Boundary Conditions

- Edges of the grid require special handling.

- **Example first row:**

- `dp[0][c]`

- Only left neighbor exists.

- **Example first column:**

- `dp[r][0]`

- Only top neighbor exists.

- Without these rules, the algorithm breaks.

---

## 🔟 Why Grid DP Is Efficient

- Without DP, we might try every possible path.

- **Example:**

- **right / down choices:**

- Number of paths grows exponentially.

- DP avoids recomputation by storing results.

- So complexity becomes:

- `O(rows × cols)`

- Instead of exponential.

---

## 🧠 Grid DP Mental Model

- Students should think like this:

- **Every cell = small problem:**
- **Neighbors = smaller solved problems:**
- **DP table = memory of answers:**

- **Algorithm becomes:**

- **1️⃣ define state:**
- **2️⃣ find transition:**
- **3️⃣ set base cases:**
- **4️⃣ fill grid in correct order:**
