# 🟢 LEVEL 6 — Dynamic Programming Foundations

---

## Concepts

- Dynamic Programming (DP) is a way to solve big problems by remembering solutions to smaller problems.
- **The main idea is very simple:**
- If we solve the same small problem many times, we should remember the answer instead of solving it again.

---

## 1️⃣ Why Dynamic Programming Exists

- **Imagine a staircase:**
- Step 3
- Step 2
- Step 1
- Ground
- **You want to reach step 3. You can move:**
- 1 step
- 2 steps
- **Question:**
- How many ways can you reach step 3?



---

## Let's think manually

- **Ways to reach step 3:**
- 1 + 1 + 1
- 1 + 2
- 2 + 1
- **Answer:**
- 3 ways
- **Now imagine step 50:**
- Trying every path becomes impossible.
- So we need a smarter way.
- That smarter method is Dynamic Programming.

---

## 2️⃣ The Key Observation

- **To reach step 3, we must come from:**
- Step 2
- or
- Step 1
- **So:**
- $ways(3) = ways(2) + ways(1)$
- **This idea is called a recurrence relation. It means:**
- The answer for a big problem depends on answers to smaller problems.

---

## 3️⃣ Overlapping Subproblems

- **Sometimes the same problem appears many times. Example recursion tree:**

> ```text
> ways(5)
>  ├─ ways(4)
>  │   ├─ ways(3)
>  │   │   ├─ ways(2)
>  │   │   └─ ways(1)
>  │   └─ ways(2)
>  └─ ways(3)
>      ├─ ways(2)
>      └─ ways(1)
> ```

- **Look carefully:**
- $ways(2)$
- $ways(3)$
- appear many times. This means we are repeating work.



---

## 4️⃣ The Solution: Remember Answers

- **Instead of recomputing $ways(3)$ again and again, we store it. Example table:**

| Step | Ways |
| :--- | :--- |
| 1 | 1 |
| 2 | 2 |
| 3 | 3 |
| 4 | 5 |
| 5 | 8 |

- Now every result is computed once. This technique is Dynamic Programming.

---

## 5️⃣ Memoization

- **Memoization means:**
- Store answers of subproblems so they can be reused.
- **Example:**

```cpp
int dp[100];
```

- **When we compute $ways(5)$, we save the result.**
- Next time we need it, we read it from memory.

---

## 6️⃣ State (The Most Important Idea)

- **In DP we define something called a state. A state describes:**
- What information we store for each subproblem.
- **Example:**
- $dp[i]$ = number of ways to reach step $i$
- **Here:**
- $i$ is the state.

---

## 7️⃣ State Transition

- **The rule connecting states is called transition. Example:**
- $dp[i] = dp[i-1] + dp[i-2]$
- **Meaning: Ways to reach step $i$ come from:**
- one step before
- two steps before

---

## 8️⃣ Base Case

- **DP must start somewhere. These starting answers are called base cases. Example:**
- $dp[1] = 1$
- $dp[2] = 2$
- Without base cases, nothing can be computed.

---

## 9️⃣ DP Table

- **We usually store answers in a table. Example:**
- $dp[0] = 1$
- $dp[1] = 1$
- $dp[2] = 2$
- $dp[3] = 3$
- $dp[4] = 5$
- $dp[5] = 8$
- Each new value uses previous values.

---

## 🔟 Bottom-Up DP

- **Instead of recursion, we can fill the table step by step. Example:**

```cpp
dp[1] = 1;
dp[2] = 2;

for(int i=3;i<=n;i++)
{
    dp[i] = dp[i-1] + dp[i-2];
}
```

- We build answers from smallest to largest.

---

## 1️⃣1️⃣ Why DP is Powerful

- **Without DP:**
- We recompute subproblems repeatedly.
- **Example complexity:**
- $O(2^n)$
- Very slow.
- **With DP:**
- Each problem solved once.
- **Complexity becomes:**
- $O(n)$
- Huge improvement.

---

## 1️⃣2️⃣ DP Mental Model

- **When solving DP problems always ask:**
- 1️⃣ What is the state?
- 2️⃣ What is the transition?
- 3️⃣ What is the base case?
- 4️⃣ What order should we compute?
- If these four are clear, the DP solution becomes simple.

---

## 🧠 Simple DP Example Summary

- **Problem:**
- Climb stairs.
- **State:**
- $dp[i]$
- **Meaning:**
- Ways to reach step $i$.
- **Transition:**
- $dp[i] = dp[i-1] + dp[i-2]$
- **Base cases:**
- $dp[1] = 1$
- $dp[2] = 2$
- **Compute from:**
- 1 → 2 → 3 → ... → n

---

## 🧭 Why This Level Matters for Extra DP Day 2

- Grid DP problems are just DP applied to a matrix.
- **Instead of $dp[i]$, we use:**
- $dp[row][col]$
- Each cell becomes a state.
- Transitions depend on neighbor cells.
